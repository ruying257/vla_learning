# TE deploy 实验脚本说明

本文档说明 `scripts/run_te_experiments.py` 的实验矩阵和运行方式。该脚本只负责使用已训练好的 checkpoint 做 temporal ensemble 参数 deploy sweep，不负责训练。

## 结论先行

- 训练由 `3.train_finetune.py` 独立完成，runner 只做 deploy。
- 当前实验设计是在同一个 checkpoint 上比较不同 temporal ensemble 参数。
- 保留 `none` 作为无 temporal ensemble 对照；脚本只包含固定 TE 参数 sweep，不包含额外实验分组开关或训练逻辑。
- `TE_none` 默认 `ACT_N_ACTION_STEPS=ACT_CHUNK_SIZE`，按完整 action chunk 执行；其它固定 TE 实验默认 `ACT_N_ACTION_STEPS=1`，每步重预测并做时间集成。

## 快速命令

查看实验矩阵：

```bash
python scripts/run_te_experiments.py --list
```

只打印 deploy 命令，不启动 MuJoCo：

```bash
python scripts/run_te_experiments.py --dry-run
```

用默认 checkpoint 跑 5 个 seed：

```bash
python scripts/run_te_experiments.py --deploy-seed-start 1 --deploy-trials 5
```

如果 checkpoint 或数据集路径不同：

```bash
python scripts/run_te_experiments.py \
  --ckpt-dir ckpt/v5_finetune_new_data \
  --dataset-root failure_seed_data
```

如果批量 deploy 中某些 seed 因 MuJoCo 初始化异常失败，可以只补跑指定 seed：

```bash
python scripts/rerun_deploy_seeds.py --exp TE_090 --deploy-seeds 3
python scripts/rerun_deploy_seeds.py --exp TE_090 --deploy-seeds 3,7,11 --continue-on-fail
python scripts/rerun_deploy_seeds.py --exp TE_001 TE_090 --deploy-seeds 3 --dry-run
```

补跑脚本要求显式传 `--exp`，只运行 deploy；成功写出 metrics 的 seed 会增量更新对应实验目录下的 `seed_results.csv`，并刷新全局 `result.csv`。

`SimpleEnv.reset(seed=...)` 会在采样前清空 MuJoCo 的动力学状态，并拒绝杯盘网格接触或稳定后越界的候选坐标。同一 seed 的重采样序列可复现；首组坐标合法时保持历史位置，旧坐标已接触时会确定性地改用下一组合法位置。

## TE 实验矩阵

| 实验 | 含义 | 当前状态 |
| --- | --- | --- |
| `TE_none` | 无 temporal ensemble | 验证 TE-off 对照 |
| `TE_001` | 固定时间集成，系数 0.01 | 接近原始 ACT 推荐的轻量平滑 |
| `TE_005` | 固定时间集成，系数 0.05 | 轻平滑 sweep |
| `TE_010` | 固定时间集成，系数 0.10 | 中低强度平滑 |
| `TE_015` | 固定时间集成，系数 0.15 | 细化 0.10 到 0.30 区间 |
| `TE_020` | 固定时间集成，系数 0.20 | 细化 0.10 到 0.30 区间 |
| `TE_030` | 固定时间集成，系数 0.30 | 低系数 sweep 上界 |
| `TE_070` | 固定时间集成，系数 0.70 | 强平滑对照 |
| `TE_090` | 固定时间集成，系数 0.90 | 当前强平滑基线 |

默认 checkpoint 为 `./ckpt/v5_finetune_new_data`，默认数据集标识为 `failure_seed_data`，均可通过命令行覆盖。

## 输出文件

所有 TE deploy 实验输出到：

```text
experiments/te_sweep/
```

主要文件包括：

- `<exp_id>/logs/`：单个实验的部署日志。
- `<exp_id>/metrics/`：单个实验内每个 seed 的部署指标。
- `<exp_id>/videos/`：单个实验的部署视频。
- `<exp_id>/seed_results.csv`：单个实验内按 seed 增量汇总的关键部署指标。
- `<exp_id>/result.csv`：单个实验的一行部署汇总。
- `result.csv`：所有 TE 实验共享的部署汇总表。

示例结构：

```text
experiments/te_sweep/TE_090/
  logs/
    deploy_seed1.log
  metrics/
    deploy_seed1.json
  result.csv
  seed_results.csv
  videos/
```

结果表字段包含：

```text
exp_id / dataset / ckpt_dir / temporal_ensemble_coeff /
success_rate /
avg_steps / avg_success_steps / action_smoothness_mean /
prediction_inconsistency_mean / video_path / notes
```

`seed_results.csv` 每一行对应一个 deploy seed，字段包含：

```text
seed / executed_steps / success / error / action_smoothness_mean /
action_smoothness_max / prediction_inconsistency_mean /
prediction_inconsistency_max
```

`seed_results.csv` 只在实际 deploy 写出 metrics 后按 seed 增量更新：补跑单个 seed 时会替换该 seed 行，新 seed 会追加到表中，未重跑或未产生 metrics 的 seed 保持不变。单实验 `result.csv` 和全局 `result.csv` 仍会基于当前可读取的 metrics 汇总更新。

`--summarize-only` 只刷新单实验和全局 `result.csv`，不会改动 `seed_results.csv`，避免误刷新未重跑的 seed 行。
刷新结果时会按当前字段定义重写 CSV；旧结果中已经删除的历史列会被自动丢弃。

## 提取固定 seed 子集

如需从已有 TE sweep 中提取相同的 seed 范围做公平对比，可运行：

```bash
python scripts/extract_te_seed_subset.py
```

默认从 `experiments/te_sweep_v5` 提取 seed 1–20，并写入
`experiments/te_sweep_v5_20seeds`。每个 TE 实验只复制对应的 metrics JSON，
不会重复复制日志和视频；脚本会基于这 20 组 JSON 重新生成实验内的
`seed_results.csv`、`result.csv` 和根目录的 `result.csv`。结果中的
`video_path` 仍指向源实验视频，便于回溯。

可通过参数指定其它范围或目录：

```bash
python scripts/extract_te_seed_subset.py \
  --source-dir experiments/te_sweep_v5 \
  --output-dir experiments/te_sweep_v5_20seeds \
  --seed-start 1 \
  --seed-end 20
```

脚本默认拒绝覆盖已有输出目录；确认需要重新生成时显式添加 `--overwrite`。
如果任一实验缺少目标 seed，或 JSON 内的 seed 与文件名不一致，脚本会在写入前报错，
避免产生不完整的对比结果。

如果某些 seed 因初始化状态异常需要从所有实验中统一排除，可使用：

```bash
python scripts/extract_te_seed_subset.py --exclude-seeds 17 --overwrite
```

多个 seed 使用逗号分隔，例如 `--exclude-seeds 17,23`。排除后，metrics、实验内
`seed_results.csv`、实验内 `result.csv` 和全局 `result.csv` 会一起重新生成。

`seed_results.csv` 的 `error` 允许人工复核后填写。使用 `--overwrite` 重建时，脚本
会优先保留目标目录已有的非空 `error`；目标值为空时，再从源目录对应实验的
`seed_results.csv` 恢复。只有两处都没有人工标注时，才使用 metrics JSON 中的值。
其它指标始终以 metrics JSON 为准。

## 从已有实验中删除异常 seed

如果某个 seed 的初始化状态会直接满足成功条件，可先检查将要删除的文件：

```bash
python scripts/remove_te_seed_data.py --seed 17 --dry-run
```

确认后执行删除：

```bash
python scripts/remove_te_seed_data.py --seed 17
```

脚本默认处理 `experiments/te_sweep_v5_finetune` 下的全部 TE 实验，并同步删除目标
seed 的 `seed_results.csv` 行、metrics、日志和视频，再基于剩余 metrics 更新实验内
及全局 `result.csv`。删除 CSV 行时不会重新序列化 `seed_results.csv`，因此其它 seed
的原始字段、人工 `error` 和换行格式保持不变。汇总表保留执行前已有的表头格式。

## 论文表述边界

- 当前 runner 不再执行训练；训练过程和 WandB 记录应引用 `3.train_finetune.py`。
- 当前 runner 不包含其它 TE 策略、关键阶段重采样或训练流程。
- deploy 结论应写成：同一个 checkpoint 在不同 temporal ensemble 参数下的闭环表现对比。

结论：可以把其他领域 TE 的思想借到 ACT 上，但不要说成“发明 Temporal Ensemble”。更合适的表述是：**针对 ACT 中 action chunk 重叠预测不一致、动作抖动、响应滞后和失败阶段不稳定等问题，设计更适合闭环机械臂控制的 TE 改进策略。**

下面给你 4 个比较适合写进 CAC 小论文的创新点。

**1. 不确定性自适应 TE**

借鉴来源：Mean Teacher、半监督学习里的 prediction confidence / consistency。

问题：
ACT 当前 TE 通常使用固定系数，比如 `0.9`。但机械臂不同阶段对平滑的需求不一样：

- 接近杯子时，需要响应快；
- 抓取和放置时，需要稳定；
- 如果模型预测分歧很大，强行平均可能会把动作拖慢；
- 如果预测很一致，强平滑可以减少抖动。

解决方法：
计算多个 action chunk 对当前动作预测的分歧，例如：

```
uncertainty_t = mean(||a_t^i - mean(a_t)||)
```

然后动态调整 TE 系数：

```
alpha_t = alpha_min + (alpha_max - alpha_min) * exp(-lambda * uncertainty_t)
```

直觉是：

- 预测一致：`uncertainty` 小，`alpha` 大，多平滑；
- 预测冲突：`uncertainty` 大，`alpha` 小，更相信当前预测。

ACT 改法：
部署阶段不用固定 `ACT_TEMPORAL_ENSEMBLE_COEFF=0.9`，而是根据 chunk 内预测不一致性动态调节。

实验设计：

- ACT 原始无 TE；
- 固定 TE：`0.7`、`0.9`；
- 自适应 TE；
- 指标：成功率、动作平滑度、预测不一致性、平均完成步数。

论文价值：
这个创新点比较自然，因为它直接回答：**为什么同一个 TE 系数不能适合所有控制阶段？**

------

**2. 阶段感知 TE**

借鉴来源：MPC / 机器人控制里的 phase-dependent controller。

问题：
抓取放置任务不是一个均匀过程，而是分阶段的：

```
接近杯子 -> 闭合夹爪 -> 搬运 -> 对准盘子 -> 释放 -> 撤离
```

不同阶段的动作需求不同：

- 接近阶段：需要快速纠偏；
- 搬运阶段：需要轨迹平滑；
- 放置阶段：需要低抖动；
- 释放阶段：不能过度平滑，否则夹爪可能延迟打开。

固定 TE 可能导致一个问题：
**放置已经完成，但夹爪动作被平滑拖慢，导致不松开。**

解决方法：
根据任务状态切换 TE 系数：

```
approach: alpha = 0.6
transport: alpha = 0.9
place: alpha = 0.8
release: alpha = 0.3
```

阶段可以不用训练新模型，先用规则判断：

- 杯子和夹爪距离；
- 杯子和盘子距离；
- 夹爪开合状态；
- 末端高度。

ACT 改法：
在 `4.deploy.py` 部署阶段增加 phase detector，然后按阶段使用不同 temporal ensemble coefficient。

实验设计：

- 固定 TE；
- 阶段感知 TE；
- 对比“夹住杯子但不松开”的失败案例数量；
- 指标：strict success、placement success、final gripper qpos、平均完成步数。

论文价值：
这个创新点非常贴合你当前观察到的问题：**杯子已经到盘子上，但夹爪不松开**。它比泛泛调参更像“从失败现象出发的改进”。

------

**3. 关键动作维度分组 TE**

借鉴来源：信号处理里的多通道滤波、机器人控制中的不同控制量不同增益。

问题：
ACT 的动作是 7 维：

```
x, y, z, roll, pitch, yaw, gripper
```

但这些维度不应该用同一个平滑强度。

例如：

- 末端位置可以适度平滑；
- 姿态可以更强平滑；
- 夹爪动作不应该被过度平滑；
- z 方向在放置阶段也不能太滞后。

如果所有动作维度统一 TE，可能出现：

```
位置已经到位，但 gripper open 被平均掉了
```

解决方法：
对不同动作维度使用不同 TE 系数：

```
alpha_pos = 0.85
alpha_rot = 0.90
alpha_gripper = 0.20
```

或者更细：

```
[x, y]: 0.85
[z]: 0.70
[rotation]: 0.90
[gripper]: 0.10
```

ACT 改法：
原来 TE 是对整个 action vector 做加权平均，现在改成逐维度加权平均。

实验设计：

- 固定整体 TE；
- 分组 TE；
- 分组 TE + release 阶段降低 gripper 平滑；
- 指标：strict success、final gripper qpos、动作平滑度、夹爪释放成功率。

论文价值：
这个点很具体，也很容易解释：**控制量的物理意义不同，所以不应该共享同一个时间平滑系数。**

------

**4. 历史一致性约束 TE**

借鉴来源：半监督学习 Temporal Ensembling、Mean Teacher 中的 consistency regularization。

问题：
ACT 训练时主要学习专家动作，但部署时的问题常常来自 chunk 之间预测不一致：

```
第 t 步预测未来动作 A
第 t+1 步又预测同一未来时刻动作 B
A 和 B 差很多
```

部署阶段 TE 只是事后平均，但模型本身没有被训练成“跨时间预测一致”。

解决方法：
训练时加入一个轻量 consistency loss，让相邻时刻预测的重叠未来动作尽量一致：

```
L_total = L_action + beta * L_consistency
```

其中：

```
L_consistency = mean(||chunk_t[k+1:] - chunk_t+1[k:]||)
```

直觉是：
如果两个相邻 observation 对未来同一段动作的预测差异太大，说明模型时间一致性差，部署时就容易抖动。

ACT 改法：
训练阶段多加一个损失项，不改 ACT 主干结构。模型仍然是 LeRobot ACT，只是在 loss 上增强时间一致性。

实验设计：

- 原始 ACT；
- ACT + 部署 TE；
- ACT + consistency loss；
- ACT + consistency loss + TE；
- 指标：mean action error、prediction inconsistency、动作平滑度、成功率。

论文价值：
这个点比纯部署技巧更像“训练优化”。它能体现你不是只会调参数，而是在思考：**为什么模型部署时会抖，能不能在训练阶段让它学得更一致？**

------

我建议优先级是：

1. **阶段感知 TE**：最贴合你的失败现象，工程实现也最小。
2. **关键动作维度分组 TE**：非常适合解决夹爪不松开问题。
3. **不确定性自适应 TE**：论文味更强，但要计算预测分歧。
4. **历史一致性约束 TE**：最像训练方法创新，但实现和实验成本最高。

如果你想发 CAC，最稳的组合是：

> **阶段感知 + 动作维度分组 TE**

因为它们都从机械臂任务本身出发，问题明确，改动可控，实验也容易解释。

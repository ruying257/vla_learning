# ACT 项目 CAC 论文创新方案

本文档面向 CAC 论文准备，目标是在现有 ACT 机械臂抓取放置项目基础上，设计两个创新性适中、实现成本可控、但能体现训练优化能力的研究方案。

核心目标不是提出复杂新模型，而是围绕任务成功率建立完整闭环：发现失败问题，设计轻量模块，做可复现实验，用闭环成功率和视频证据证明方法有效。

## 总体选题定位

研究对象：

- 任务：MuJoCo 中 UR 系列机械臂将杯子放置到盘子上。
- 策略：基于 LeRobot 的 ACT 模仿学习策略。
- 输入：第三方视角图像、手腕视角图像、末端状态。
- 输出：7 维动作。
- 评价目标：提高闭环任务成功率，而不是只降低训练 loss。

从第一性原理看，ACT 部署失败主要来自三个层面：

1. 模型预测不稳定：同一时刻来自不同 action chunk 的动作预测不一致，容易造成动作抖动或轨迹漂移。
2. 数据分布不足：模仿学习只能学习示范覆盖到的状态，一旦闭环执行偏离示范分布，策略缺少纠偏经验。
3. 离线指标和闭环效果不完全一致：loss 降低不一定代表任务成功率提升，必须通过部署视频和成功率验证。

因此推荐两条轻量论文方案：

- 方案一：预测一致性驱动的自适应时间集成 ACT。
- 方案三：失败样例引导的数据增强 ACT。

两者可以单独成文，也可以组合成一个完整框架：前者优化部署阶段动作稳定性，后者优化训练数据分布覆盖。

## 方案一：预测一致性驱动的自适应时间集成 ACT

### 1. 基于问题

ACT 通过 action chunking 一次预测未来一段动作。部署时常用 temporal ensemble 将多个历史 action chunk 对当前时刻的预测加权平均，以降低动作抖动。

现有问题是 temporal ensemble 的平滑系数通常固定，例如 `temporal_ensemble_coeff=0.9`。固定系数无法适应不同执行阶段：

- 系数较小：最新预测占比高，响应快，但动作容易抖动。
- 系数较大：历史预测占比高，动作平滑，但纠偏变慢。
- 接近杯子、抓取、放置阶段需要快速修正。
- 长距离移动阶段更需要稳定平滑。

进一步看，同一时刻如果多个历史 chunk 给出的动作预测差异很大，说明策略对当前动作不确定。此时继续使用固定平滑会把不一致预测混在一起，可能加剧轨迹偏移。

### 2. 解决方法

提出一个部署侧轻量模块：预测一致性驱动的自适应时间集成，英文可写作 Consistency-Aware Temporal Ensemble，简称 CATE。

核心思想：

- 收集多个历史 action chunk 对当前时刻动作的预测。
- 计算这些预测动作之间的一致性。
- 如果预测一致，说明策略稳定，提高平滑强度。
- 如果预测分歧大，说明策略正在不确定或需要纠偏，降低历史预测权重，让最新预测更快生效。

可以使用动作方差作为不一致性指标：

```text
u_t = mean_i(||a_t^i - mean(a_t)||_2)
c_t = exp(-lambda * u_t)
alpha_t = alpha_min + c_t * (alpha_max - alpha_min)
```

其中：

- `a_t^i` 表示第 `i` 个历史 action chunk 对当前时刻的预测。
- `u_t` 表示预测不一致程度。
- `c_t` 表示预测置信度。
- `alpha_t` 表示当前时刻自适应时间集成系数。

最终动作输出：

```text
a_t = TemporalEnsemble({a_t^i}, alpha_t)
```

建议参数：

| 参数 | 建议值 | 含义 |
| --- | --- | --- |
| `alpha_min` | 0.5 | 预测分歧大时保留一定平滑 |
| `alpha_max` | 0.95 | 预测一致时强平滑 |
| `lambda` | 5 到 20 | 控制一致性到置信度的映射强度 |
| `window_size` | 10 到 50 | 用于集成的历史 chunk 数量 |

### 3. 部署侧轻量模块设计

该模块位于 ACT policy 输出和 MuJoCo 环境执行动作之间。ACT 原模型仍负责根据当前观测预测 action chunk，部署侧模块只负责决定当前 step 如何融合历史 chunk 中对当前时刻的多个动作预测。

模块输入：

- 当前 step 的 policy 输入 observation。
- 当前 ACT 预测的 action chunk。
- 历史 action chunk 缓存，缓存项包含 chunk 的起始 step 和动作序列。

模块输出：

- 当前实际执行动作。
- 当前预测不一致性 `u_t`。
- 当前自适应平滑系数 `alpha_t`。

执行流程：

1. 每个部署 step 调用 ACT 预测新的 action chunk。
2. 将 chunk 与起始 step 一起存入缓存。
3. 从缓存中取出所有仍覆盖当前 step 的动作预测。
4. 计算这些预测之间的平均 L2 分歧，得到 `u_t`。
5. 将 `u_t` 映射为 `alpha_t`。预测越一致，`alpha_t` 越大，动作越平滑；预测越分歧，`alpha_t` 越小，最新预测占比越高。
6. 用 `alpha_t` 对当前可用预测做时间集成，得到最终动作并送入 MuJoCo。

从第一性原理看，这个模块利用的是 ACT action chunk 自带的多步预测冗余。它不需要额外网络，也不需要修改 loss，只把不同历史 chunk 对当前动作的“意见分歧”当作部署时置信度。

工程上可以把它做成独立部署脚本，例如 `4.deploy_cate_adaptive.py`，并用独立批量 runner `scripts/run_cate_adaptive_experiments.py` 组织 CATE-E0 到 CATE-E3 的对照实验，避免影响原有部署脚本和已有实验结果。

### 4. 实现方案

建议新增独立部署脚本，不改训练代码和现有部署代码：

- 新增动作集成函数，例如 `compute_adaptive_temporal_ensemble(...)`。
- 在 `4.deploy_cate_adaptive.py` 中保存历史 action chunk。
- 每个 step 根据当前可用预测计算 `u_t` 和 `alpha_t`。
- 将 `alpha_t`、`u_t`、动作差分写入 metrics，便于论文分析。

建议新增环境变量：

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `ACT_ADAPTIVE_TE` | `1` | 是否启用自适应时间集成；固定对照由 runner 显式设为 `0` |
| `ACT_ADAPTIVE_ALPHA_MIN` | `0.5` | 最小平滑系数 |
| `ACT_ADAPTIVE_ALPHA_MAX` | `0.95` | 最大平滑系数 |
| `ACT_ADAPTIVE_LAMBDA` | `10` | 不一致性映射系数 |
| `ACT_ADAPTIVE_WINDOW_SIZE` | `50` | 历史 chunk 缓存窗口 |

### 5. 实验方案

对照组设计：

| 实验编号 | 方法 | 说明 |
| --- | --- | --- |
| CATE-E0 | 原始 ACT，无时间集成 | 验证不平滑时的抖动问题 |
| CATE-E1b | 固定 `alpha=0.3` | 弱平滑，响应更敏感但可能更抖 |
| CATE-E1 | 固定 `alpha=0.7` | 中等平滑，响应较快 |
| CATE-E2 | 固定 `alpha=0.9` | 当前常用强平滑基线 |
| CATE-E3 | 自适应时间集成 | 本文方法 |

为了避免变量混杂，方案一建议固定 checkpoint，只比较部署策略：

```bash
ACT_CKPT_DIR=./ckpt/act_y
ACT_DATASET_ROOT=datasets/demo_v5_30demos_random
ACT_DEPLOY_MAX_STEPS=300
ACT_RECORD_VIDEO=1
```

评价指标：

| 指标 | 计算方式 | 论文意义 |
| --- | --- | --- |
| 成功率 | 20 次部署成功次数 / 20 | 最终核心指标 |
| 平均完成步数 | 成功样本完成 step 均值 | 反映执行效率 |
| 动作平滑度 | `mean(||a_t - a_{t-1}||_2)` | 衡量抖动 |
| 预测不一致性 | `mean(u_t)` | 衡量 action chunk 分歧 |
| 轨迹偏移 | 末端或杯子到目标距离 | 分析失败模式 |
| 视频证据 | 每组至少 1 个视频 | 支撑定性分析 |

推荐执行顺序：

1. 使用 5 次部署做 smoke，确认视频和 metrics 正常。
2. 每组跑 20 次闭环部署。
3. 汇总成功率、动作平滑度、平均完成步数。
4. 选取典型成功和失败视频做案例分析。

示例命令：

```bash
ACT_RECORD_VIDEO=1 ACT_DEPLOY_MAX_STEPS=300 ACT_TEMPORAL_ENSEMBLE_COEFF=0.9 python 4.deploy_headless.py
```

自适应版本实现后可使用：

```bash
ACT_RECORD_VIDEO=1 ACT_DEPLOY_MAX_STEPS=300 ACT_ADAPTIVE_TE=1 python 4.deploy_cate_adaptive.py
```

批量实验可使用：

```bash
python scripts/run_cate_adaptive_experiments.py --exp CATE_E3_adaptive --deploy-trials 20
```

### 6. 预期结果

预期趋势：

- 无时间集成：动作抖动较明显，成功率较低。
- 固定低平滑：响应快，但放置阶段容易震荡。
- 固定高平滑：动作稳定，但偏离轨迹后纠偏慢。
- 自适应时间集成：在预测一致时保持稳定，在预测分歧时提高响应速度，因此成功率更高。

可以在论文中给出以下结论：

> 自适应时间集成利用 action chunk 预测之间的一致性估计策略置信度，在不重新训练模型的情况下改善了部署阶段动作稳定性和纠偏能力。

### 7. 可行性分析

优点：

- 实现成本低，主要改部署脚本。
- 不需要重新训练 ACT。
- 能直接利用现有 headless 部署视频和 metrics。
- 论文逻辑清楚，和 ACT 的 action chunking 机制强相关。

风险：

- 如果 baseline 已经很稳定，成功率提升可能不明显。
- 方差阈值或 `lambda` 需要小范围调参。
- 只改部署后处理，创新深度有限。

规避方式：

- 同时报告动作平滑度和成功率，不只看成功率。
- 选择不同随机初始位置部署，放大闭环纠偏差异。
- 保留失败视频，解释固定平滑为什么失败。

## 方案三：失败样例引导的数据增强 ACT

### 1. 基于问题

ACT 属于模仿学习。模仿学习的本质是从专家示范分布中学习状态到动作的映射。如果训练数据只覆盖少量固定轨迹，策略在闭环部署中一旦偏离示范分布，就可能进入训练集中没见过的状态。

在杯子放置任务中，常见失败包括：

- 靠近阶段末端偏移，导致抓取点不准。
- 抓取阶段夹爪闭合时机不稳定。
- 搬运阶段杯子轨迹偏离盘子。
- 放置阶段高度或水平位置误差较大。
- 动作抖动导致物体滑落或最终状态不满足成功条件。

如果只增加随机 demo 数量，不一定能有效覆盖这些失败状态。更有效的方式是根据闭环失败结果，有针对性地补充关键阶段数据。

### 2. 解决方法

提出一个训练数据侧轻量模块：失败样例引导的数据增强，英文可写作 Failure-Guided Demonstration Augmentation，简称 FGDA。

核心思想：

1. 用 baseline ACT 进行多次闭环部署。
2. 根据视频和 metrics 标注失败类型。
3. 针对高频失败阶段补采少量专家示范。
4. 训练时提高关键阶段样本或失败相关 demo 的采样概率。
5. 重新训练 ACT，并比较闭环成功率。

该方案不是简单堆数据，而是让数据采集和训练采样围绕失败模式闭环优化。

### 3. 失败类型定义

建议建立统一失败标签：

| 标签 | 判断依据 | 对应优化方向 |
| --- | --- | --- |
| `grasp_offset` | 末端到杯子距离较大，抓取失败 | 补采靠近和抓取示范 |
| `gripper_timing` | 到达杯子附近但夹爪开合时机错误 | 补采夹爪动作边界数据 |
| `transfer_drift` | 抓住杯子后移动轨迹偏离盘子 | 补采搬运路径纠偏示范 |
| `place_overshoot` | 杯子到盘子附近但放置偏移 | 补采放置阶段精细控制 |
| `unstable_action` | 末端动作高频抖动 | 调整平滑或补采稳定轨迹 |

### 4. 数据增强方法

方法 A：失败场景补采。

针对部署失败的初始位置和相似位置，人工遥操作补采专家示范：

- 每类主要失败补采 5 到 10 条 demo。
- 优先补采随机初始位置，而不是固定初始位置。
- 每条 demo 保留完整轨迹和多视角图像。

推荐数据组织：

```text
datasets/
  demo_v5_30demos_random/
  demo_v6_failure_guided_10demos/
  demo_v7_random_plus_failure_guided/
```

方法 B：关键阶段重采样。

在训练采样时提高关键阶段样本概率：

- 靠近杯子阶段。
- 夹爪闭合前后。
- 杯子接近盘子阶段。
- 放置释放前后。

如果暂时不改 dataset 类，可以先用简单方式实现：

- 把失败引导补采 demo 复制进合并数据集。
- 在合并时让关键 demo 重复出现 2 到 3 次。
- 先验证成功率提升，再考虑实现正式 weighted sampler。

方法 C：轻量数据质量筛选。

训练前过滤明显低质量 demo：

- 轨迹中断。
- 夹爪没有成功抓住杯子。
- 最终没有满足成功条件。
- 图像帧缺失或动作维度异常。

这一步可以作为论文中的数据清洗模块，和失败引导增强共同组成数据优化流程。

### 5. 实验方案

对照组设计：

| 实验编号 | 数据集 | 训练策略 | 目的 |
| --- | --- | --- | --- |
| FGDA-E0 | 原始 30 条 random demo | 原始 ACT | baseline |
| FGDA-E1 | 原始数据 + 10 条普通 random demo | 原始 ACT | 验证单纯增加数据的效果 |
| FGDA-E2 | 原始数据 + 10 条失败引导 demo | 原始 ACT | 验证针对性补采是否更有效 |
| FGDA-E3 | 原始数据 + 失败引导 demo + 关键阶段重采样 | 原始 ACT | 验证采样优化效果 |

建议先固定模型超参数，避免数据变量和训练变量混在一起：

```bash
ACT_CHUNK_SIZE=50
ACT_BATCH_SIZE=64
ACT_LR=1e-4
ACT_TRAINING_STEPS=6000
ACT_NUM_WORKERS=0
```

评价指标：

| 指标 | 计算方式 | 论文意义 |
| --- | --- | --- |
| 训练 final loss | 最后若干 step loss 均值 | 辅助观察，不作为最终结论 |
| Mean action error | 离线动作预测误差 | 衡量模仿误差 |
| 闭环成功率 | 每组 20 次部署成功率 | 核心指标 |
| 失败类型占比 | 每类失败次数 / 总失败次数 | 证明失败模式被改善 |
| 泛化成功率 | 随机初始位置成功率 | 证明数据分布覆盖提升 |
| 平均完成步数 | 成功样本 step 均值 | 评价效率 |

推荐执行顺序：

1. 跑 baseline 部署 20 次，生成失败视频和 metrics。
2. 按失败标签统计主要失败类型。
3. 针对最高频的 1 到 2 类失败补采 demo。
4. 训练 FGDA-E1、FGDA-E2、FGDA-E3。
5. 每组部署 20 次并保存视频。
6. 对比成功率和失败类型占比变化。

### 6. 预期结果

预期趋势：

- 普通增加 demo 可以提升一部分成功率，但提升不稳定。
- 失败引导 demo 对主要失败模式改善更明显。
- 关键阶段重采样进一步降低抓取偏移或放置偏移。
- final loss 不一定最低，但闭环成功率可能最高。

论文中可以强调：

> 该结果说明 ACT 策略的闭环表现不仅取决于模型结构，也高度依赖示范数据对关键状态和纠偏动作的覆盖。失败引导的数据增强能够以较小数据增量提升策略泛化能力。

### 7. 可行性分析

优点：

- 最贴近真实机器人训练优化流程。
- 容易得到任务成功率提升。
- 能体现数据分析、失败诊断、再训练和闭环评估能力。
- 不需要改 ACT 模型主体，风险较低。

风险：

- 需要重新采集 demo，时间成本高于方案一。
- 人工标注失败类型可能带主观性。
- 如果新增 demo 质量不高，可能反而降低训练效果。

规避方式：

- 每条新 demo 都用 `2.visualize_data.py` 回放检查。
- 使用统一失败标签，减少主观描述。
- 每个数据集固定相同训练超参数。
- 不只报告 loss，必须报告成功率和失败类型变化。

## 两个方案的组合写法


论文题目可以写成：

```text
A Lightweight Closed-loop Optimization Framework for ACT-based Robotic Pick-and-Place
```

```text
面向 ACT 机械臂模仿学习的轻量闭环优化方法
```

## 论文结构建议

摘要：说明 ACT 在机械臂抓取放置中存在动作抖动、轨迹偏移和数据分布不足问题。提出一个轻量闭环优化方法，包括预测一致性自适应时间集成和失败引导数据增强。实验表明，该方法提升了 MuJoCo 任务成功率，并降低了动作抖动和主要失败类型占比。

引言：重点写 ACT 在长时序控制中的优势、闭环部署中 loss 和成功率不一致的问题、固定时间集成和数据分布不足的局限，以及本文轻量改进的意义。

方法：分为 Consistency-Aware Temporal Ensemble 和 Failure-Guided Demonstration Augmentation 两节。

实验：包含仿真环境、数据集、训练配置、对照组、评价指标、定量结果和失败案例分析。

结论：强调任务成功率提升、失败模式减少、轻量模块可与原始 ACT 兼容，并说明后续可扩展到真实机械臂或更多操作任务。

## 最小落地路线

建议按以下顺序执行：

1. 先实现方案一，因为不需要重新采集数据。
2. 用 headless 部署批量跑 20 次，确认成功率和平滑指标。
3. 根据失败视频统计主要失败类型。
4. 补采 10 条失败引导 demo。
5. 重新训练 ACT，验证方案三。
6. 最后组合方案一和方案三，形成论文主实验。
